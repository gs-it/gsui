CSS 가이드

1. 기본 정책

  • CSS문서 인코딩과 문자 인코딩은 UTF-8을 원칙으로 한다.
  • 모든 속성(property)은 소문자로 작성한다.
  • 따옴표는 생략 가능하나 사용시 반드시 홀따옴표만 사용한다.
  • 초기화 선언(CSS Reset) 스타일은 문서의 최상단에 위치한다.
  • !important 필터는 사용하지 않는다.
  • CSS코드 작성시 개행하지 않는다.
  • 빈 줄은 1줄 이상 사용하지 않는다. 빈 줄 사용은 강제 요구사항이 아니다.
/* 인코딩 선언 */
@charset "utf-8"
/* 소문자, 홀따옴표 사용 */
.tit {overflow:hidden;width:940px;font-family:'Malgun gothic',dotum;}
/* 개행 불가 */
.list {
	float:left;
	margin-left:10px;} (X)
/* !important 사용 불가 */
.al {text-align:center !important;}(X)

2. CSS문서 선언

  • External 방식 - 스타일 내용을 CSS 파일로 따로 저장하여 페이지에 링크시키는 방법. 이 방법을 권장 사용한다.
  • Internal 방식 - head 태그 또는 body 태그 안에 style 태그로 선언 하는 방법. 이벤트등 일시적 페이지에서 사용 가능.
  • Inline 방식 - HTML 태그에 직접 스타일 값을 지정하는 방법.
  • Import 방식은 사용을 지양한다.

2. 선택자(Selector)

  • 공통 선택자 '*' 는 웹페이지 성능을 저하시키고, 주석에 영향을 미칠수 있으므로 사용하지 않는다.
  • id 선택자의 사용은 미리 예약한 레이아웃 등에서만 사용하고 되도록 사용을 지양한다.
  • 예약된 선택자는 'gs-'의 접두어를 갖는다.
  • 중복 속성 클래스의 생성빈도를 낮추기위해 OOCSS(Object-Oriented Cascading Style Sheets), 객체지향 스타일시트의 작성을 권장한다.
  • 선택자 상속을 최소화 한다. 3단 이하의 선택자 상속 사용을 권장한다.
/* 공통 선택자 사용 불가 */
.body * {font-size:12px;} (X)
/* 3단 이상의 선택자 상속 지양 */
.left .menulist .menu .first {float:left;background:none;} (X)

3. 속성(Property)

  • 속성의 선언은 레이아웃 관련 속성을 우선 선언한다.
  • 속성중 축약이 가능한 속성은 축약하여 작성한다.
  • 속성값이 0인 경우 단위는 쓰지 않는다.
  • z-index 속성값은 최대 5000이며, 하단의 z-index 수치표를 기반으로 작성한다.
/* 레이아웃 관련 속성을 우선 선언 */
.ex1 {display:block;font-size:12px;}
/* 축약 가능한 속성값은 축약을 권장 */
.ex2 {padding:0 10px;font:12px/1.4em 'Malgun Gothic';}

3. 주석

  • 공동작업 및 유지보수의 용이함을 위해 주석(/* */)을 권장한다.
  • 특별한 경우를 제외하고, /*+공백+주석내용+공백+*/ 의 형식을 유지한다.

4. 핵(CSS Hack)

  • 크로스 브라우징을 위한 CSS 핵을 허용한다.
  • GS Shop의 배포도구는 스타핵(*) 하나만 허용한다. 그 외의 핵은 전용 CSS 파일로 관리 한다.
  • CSS 핵은 개행을 허용한다.
