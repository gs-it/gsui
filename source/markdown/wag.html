Web Accessibility Guide

장애인 차별 금지법 시행에 따른 웹접근성(web accessibility) 체크 리스트

원칙

원칙 1. 인식의 용이성(Perceivable):
모든 콘텐츠는 사용자가 인식할 수 있어야 한다.
  1. 1.1 (대체 텍스트) 텍스트 아닌 콘텐츠에는 대체 텍스트를 제공해야 한다.
  2. 1.2 (멀티미디어 대체 수단) 동영상, 음성등 멀티미디어 콘텐츠를 이해할 수 있도록 대체 수단을 제공해야 한다.
  3. 1.3 (명료성) 콘텐츠는 명확하게 전달되어야 한다.
  1. 1.1.1 (적절한 대체 텍스트 제공) 텍스트 아닌 콘텐츠는 그 의미나 용도를 이해할 수 있도록 대체 텍스트 를 제공해야 한다.
  2. 1.2.1 (자막제공) 멀티미디어 콘텐츠에는 자막, 원고 또는 수화를 제공해야 한다.
  3. 1.3.1 (색에 무관한 콘텐츠 인식) 콘텐츠는 색에 관계 없이 인식될 수 있어야 한다.
  4. 1.3.2 (명확한 지시 사항 제공) 지시 사항은 모양, 크기, 위치, 방향, 색, 소리 등에 관계없이 인식될 수 있어야 한다.
  5. 1.3.3 (텍스트 콘텐츠의 명도 대비) 텍스트 콘텐츠와 배경 간의 명도 대비는 4.5대 1 이상이어야 한다.
  6. 1.3.4 (배경음 사용 금지) 자동으로 생성되는 배경음을 사용하지 않아야 한다.
원칙 2. 운용의 용이성(Operable): 사용자 인터페이스 구성 요소는 조작 가능하고 내비게이션할 수 있어야 한다.
  1. 2.1 (키보드 접근성) 콘텐츠는 키보드로 접근할 수 있어야 한다.
  2. 2.2 (충분한 시간 제공) 콘텐츠를 읽고 사용하는 데 충분한 시간을 제공해야 한다
  3. 2.3 (광과민성 발작 예방) 광과민성발작을 일으킬 수 있는 콘텐츠를 제공하지 않아야 한다.
  4. 2.4 (쉬운 내비게이션) 콘텐츠는 쉽게 내비게이션 할 수 있어야 한다.
  1. 2.1.1 (키보드 사용 보장) 모든 기능은 키보드만으로도 사용할 수 있어야 한다.
  2. 2.1.2 (초점 이동) 키보드에 의한 초점은 논리적으로 이동해야 하며 시각적으로 구별할 수 있어야 한다
  3. 2.2.1 (응답 시간 조절) 시간 제한이 있는 콘텐츠는 응답시간을 조절할 수 있어야 한다.
  4. 2.2.2 (정지 기능 제공) 자동으로 변경되는 콘텐츠는 움직임을 제어할 수 있어야 한다.
  5. 2.3.1 (깜빡임과 번쩍임 사용 제한) 초당 3~50회 주기로 깜빡이거나 번쩍이는 콘텐츠를 제공하지 않아야 한다.
  6. 2.4.1 (반복 영역 건너뛰기) 콘텐츠의 반복되는 영역은 건너뛸 수 있어야 한다.
  7. 2.4.2 (제목 제공) 페이지, 프레임, 콘텐츠 블록에는 적절한 제목을 제공해야 한다.
  8. 2.4.3 (적절한 링크 텍스트) 링크 텍스트는 용도나 목적을 이해 할 수 있도록 제공해야 한다.
원칙 3. 이해의 용이성(Understandable): 콘텐츠는 이해할 수 있어야 한다.
  1. 3.1 (가독성) 콘텐츠는 읽고 이해하기 쉬워야 한다.
  2. 3.2 (예측 가능성) 콘텐츠의 기능과 실행결과는 예측 가능해야 한다.
  3. 3.3 (콘텐츠의 논리성) 콘텐츠는 논리적으로 구성해야 한다.
  4. 3.4 (입력 도움) 입력 오류를 방지하거나 정정할 수 있어야 한다.
  1. 3.1.1 (기본 언어 표시) 주로 사용하는 언어를 명시해야 한다.
  2. 3.2.1 (사용자 요구에 따른 실행) 사용자가 의도하지 않은 기능(새 창, 초점 변화 등)은 실행되지 않아야 한다.
  3. 3.3.1 (콘텐츠의 선형화) 콘텐츠는 논리적인 순서로 제공해야 한다.
  4. 3.3.2 (표의 구성) 표는 이해하기 쉽게 구성해야 한다.
  5. 3.4.1 (레이블 제공) 입력 서식에는 대응하는 레이블을 제공 해야 한다.
  6. 3.4.2 (오류 정정) 입력 오류를 정정할 수 있는 방법을 제공 해야 한다.
원칙 4. 견고성(Robust): 웹 콘텐츠는 미래의 기술로도 접근할 수 있도록 견고하게 만들어야 한다.
  1. 4.1 (문법 준수) 웹 콘텐츠는 마크업 언어의 문법을 준수해야 한다.
  2. 4.2 (웹 애플리케이션 접근성) 웹 애플리케이션은 접근성이 있어야 한다.
  1. 4.1.1 (마크업 오류 방지) 마크업 언어의 요소는 열고 닫음, 중첩 관계 및 속성 선언에 오류가 없어야 한다.
  2. 4.2.1 (웹 애플리케이션 접근성 준수) 콘텐츠에 포함된 웹 애플리케이션은 접근성이 있어야 한다.
