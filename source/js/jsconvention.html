1. 코딩 표준화

코딩 표준화의 필요성

  • 소프트웨어에 대한 비용의 80% 정도가 유지 보수에 지출된다.
  • 어떤 소프트웨어도 저작권자가 끝까지 관리하지는 않는다.
  • 코딩 규칙은 소프트웨어의 가독성을 증대시키고 엔지니어들이 새로운 코드를 보다 빠르게 이해하도록 한다.
  • 소스 코드를 제품으로 배포하려는 경우에는, 그 소스 코드가 잘 묶어야 할 뿐만 아니라 깨끗해야 한다.

코딩 표준화의 방향

  • 코딩 표준화는 기본적으로 이해하기 쉬운 코드와 개발과 관리가 용이한 코드의 생산을 통해 전체적 개발 기간과 비용의 절감을 목표로 한다.

2. 자바스크립트 코딩 규칙

file

  • Javascript 파일의 확장자는 .js로 작성한다. (ex: xxx.js)
  • HTML과 분리시켜 관리 한다.
  • 스크립트 태그는 되도록 body 태그 최하단에 위치 시킨다.

들여쓰기

  • 1개의 탭(Tab)은 4개의 스페이스를 기본으로 한다.
  • 파일 배포전에는 반드시 코드 최적화(JS Lint)를 진행한다.

줄 길이

  • 기본적으로 줄 길이가 길어지면 가독성이 떨어지기 때문에 80자 이내를 준수 한다.
  • 줄바꿈은 반드시 콤마 이후에 실행한다.

변수선언

  • 변수의 이름은 카멜케이스(Camel Case)를 적용하고 상수로 써야 될 변수의 이름은 대문자로 하되 단어와 단어 사이에 언더 스코어('_')를 사용한다.
  • 변수 선언 후 주석으로 이 변수의 쓰임을 알려준다.
  • 데이터 형태에 따라 변수 앞에 의미에 맞는 이름을 넣어주는 것도 변수의 형태를 알려주는 방법이기도 하다.
// 카멜케이스
var mainBanner = '';
// 상수 표기
var MAX_HEIGHT = '1000'; // 상수

// 변수 선언후 주석 표기
var mainBanner = new MainBanner('#banner'); // 메인 에서 사용하는 배너

// 의미에 맞는 변수명
var arrPosX = []; // position X 의 배열
var objPageInfo = {}; // 페이지 정보 오브젝트

함수선언

  • 함수의 이름은 카멜케이스(Camel Case)를 적용한다.
  • 선언적 함수
  • 익명 함수
// 선언적 함수
function myFunc() {
var output = prompt('숫자 입력', '숫자');
alert(output);
}
myFunc();

// 익명 함수
(function(){
var output = prompt('숫자 입력', '숫자');
alert(output);
})();

3. 자바스크립트 최적화 방법

스크립트의 위치

  • 브라우저는 화면해석을 할때 스크립트 태그를 만나면 화면해석을 중지하고 스크립트 부터 해석을 한다. 만약 화면해석보다 먼저 선행되어야 한다면 head태그에 위치시켜야 하지만 웹 페이지의 속도향상의 측면에서는 body 맨 아래에 위치하는 것이 좋다. 

	...

스크립트 묶기

  • 외부의 스크립트 파일을 로드할경우 1KB 파일 4개보다 4KB 파일 1개를 더 빨리 로드 한다.
  • 여러 스크립트 파일을 하나의 파일로 앞축하여 배포한다.

지연 스크립트

  • 태그의 defer 속성을 이용하여 파일로드를 지연 시킨다.
  • IE4 이상 fireFox3.5 이상에서만 동작

스크립트 동적로딩

  • 페이지 내에 동적으로 스크립트를 추가한다.
var scriptEl = document.createElement('script');
scriptEl.type = 'text/javascript';
scriptEl.src = 'example.js';
document.getElementsByTagName('head')[0].appendChild(scriptEl);
