HTML Principles

개요

  • HTML작성시 W3C의 웹표준 권고안(Recommendation)을 기반으로 아래의 규칙을 준수하여 작성한다.
  • 웹표준을 준수하고 크로스브라우징을 구현해 서로 다른 장비에 따른 차별을 하지 않는다.
  • 웹접근성(모든 사용자가 신체적, 환경적 조건에 관계없이 접근하여 이용할 수 있도록 보장)을 확보 한다.
  • 구조적 문서 작성 방식을 따라 의미있는 태그를 사용하고 구조와 표현을 분리한다.

정의

DOCTYPE 정의

  • 표준 모드의 문서타입을 선언함으로써 모든 브라우저에서 일관된 렌더링을 구현할 수 있다.
  • 다양한 플랫폼 지원을 위해 XHTML 1.0, 하위 브라우져 호환성을 위해 Transitional 모드를 사용한다.
  • HTML4 문서타입과 HTML5 문서타입이 혼재 되어있으나, 기술적 제약이 없는 한 HTML5 문서타입 사용을 권장한다.

문자열의 인코딩 선언

  • W3C에서는 HTML로 표시되는 HTML 및 XHTML 파일은 Content-Type 태그를 포함하고 UTF-8 사용을 권장한다.
  • 아래 하나의 문자열 인코딩 선언을 사용한다.

문서제목의 표현

  • 문서의 제목을 의미에 맞게 사용함으로써웹표준, 웹접근성에 기여한다.
  • 규칙을 일원화해 서비스의 통일성을 높이고 검색 엔진에 대한 최적화를 꾀한다.
  • 서비스명이 우선하며 서비스외 페이지는 기본 문서 제목을 따른다.

Meta Title

  • 검색엔진 최적화(SEO)를 위해 Meta요소를 이용하여 문서 제목을 추가 명시한다.
  • 인터넷 익스플로러의 버전 관리를 위해 이를 위한 Meta 태그를 명시한다.
  • 단품 페이지의 경우 해당 상품의 정보를 추가한다.

일반 규칙

기본 문법

  • 모든 요소는 완벽한 마크업(Mark Up - 열고/닫기) 구조를 유지한다.
  • 비표준 엘리먼트( font, center, strike, u, s, applet, basefont, dir, isindex, xmp )를 사용하지 않는다.
  • 모든 속성값은 속성이 함께 선언되어야 한다.
  • 모든 요소와 속성은 소문자여야 한다.
  • 모든 속성 값은 인용 부호안에 표기한다. 인용부호는 큰 따옴표("")를 사용한다.

주석

  • HTML 코드의 주석은 코드의 수정과 삭제를 표시하거나, 개발자가 참고해야 하는 사항을 기술한다.
  • 종료 주석은 //를 표기해 구분하고 id(#)값 또는 class(.)값을 표기해 해당 요소의 종료를 표기한다.
