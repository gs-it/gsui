Epilogue

Goal

GS SHOP의 설계 및 개발시 일원화된 구조와 공통된 정책을 통해 서비스 통일성 및 개발 효율성을 확보하기 위해 UI Development Convention을 구축하고자 하였다.

As-is

  • 유사 디자인 UI템플릿의 재사용률이 낮음
  • 동일 요소의 중복된 코드 산재
  • 개발시 매번 기존 코드를 분석해야 하는 번거로움
  • 패널 프로젝트 인수인계시 기존 코드와의 병합이 어려움

To-be

운영 효율성 증대, 신규 구축시 가이드 제시 및 기술지원

  • UI/CSS/HTML 가이드를 활용함으로써 개발공수 절감 및 업무 중복성 해소
  • 단일화된 코드 및 경로 사용으로 유지보수가 용이
  • 패널 프로젝트 구축 시 가이드 제시를 통해 신규 구축과 기존 운영의 연속성 유지
  • 대내외 홍보효과 기대

Merit

GS SHOP의 브랜드, 도메인을 사용하는 모든 유무선 웹 서비스에 적용할 수 있으며, 신규 서비스의 구축 및 운영을 추진하는 각 사업부서에게 가이드 및 기술지원 역할을 할 수 있어 통일된 UI/UX를 지향할 수 있다.

Description

  • 유무선 페이지 문서타입 지정(DTD Type)
  • 이미지, CSS(id/class), JS, HTML 명명규칙 작성
  • 이미지, CSS, JS 폴더구조 설계
  • HTML 마크-업 가이드 작성
  • UI 템플릿 라이브러리 구축
  • 자주 사용하는 UI 스크립트 공통화
  • 현업에게 UI/UX 가이드 제공
  • IT 구축 실무자에게 공통 정책 가이드 제시 및 기술지원 가능
