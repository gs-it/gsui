WHY? Coding Convention

1. 필요성

  • 마크업 개발은 프론트엔드 페이지의 기본을 형성한다.
  • 마크업 개발을 잘해야 모든 브라우저에서 콘텐츠의 손실 없이, 빠르고 쉽게 이용자에게 전달할 수 있다.
  • 위의 두가지를 만족시키기위해 마크업 개발자가 지켜야할 표준이 코딩켄벤션이다.
  • 처음 코드를 작성자가 아니어도 이해할 수 있도록 작성하여 운영의 용이성을 제공한다. 이는 유지보수 비용을 절감하고 업무 효율을 높이는 역할을 한다.
  • 다수의 프로젝트 진행시 통일된 기준으로 소스 코드를 작성 할 수 있는 기준이 된다.

2. 용어

엘리먼트(element)
HTML문서를 구성하는 모든 태그요소
head, body, div, p, span, table, img, form ....
애트리뷰(attribute)
엘리먼트의 특성(애트리뷰 값은 큰따옴표("")로 묶는다)
선택자(selector)
엘리먼트에 CSS 스타일적용할 수있는 객체
*, div, div p, div > p, div#id, div.class, div:nth-child ...
속성(property)
엘리먼트 CSS 특성
font-weight(property):bold(value)
예약어
#wrap, #header, #content,...
